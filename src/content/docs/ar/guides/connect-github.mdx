يدفع خيار **إنشاء مستودع GitHub** في [Build Studio](/ar/build/) القالب المُولَّد
الخاص بك (الدستور، والمواصفات، ومُحفِّز الوكيل، والوصفة، و`package.json`) مباشرةً إلى
مستودع جديد تمامًا — دون ملف مضغوط، ودون طرفية.

<Aside type="tip" title="إنه حسابك، ومستودعك">
كل باني يربط حساب GitHub **الخاص به**. عندما تنقر على *ربط
GitHub*، تسجّل الدخول بصفتك أنت، ويُنشأ المستودع في حسابك **أنت**، ويبقى
رمز الوصول في متصفحك فقط لبضع دقائق — لا على خادمنا أبدًا،
ولا يُشارك مع أي شخص آخر. يسجّل القائم على الصيانة التطبيق مرة واحدة (الخطوة
أدناه)؛ بعد ذلك، يصرّح به كل باني لنفسه.
</Aside>

## للبانين

<Steps>

1. **أنشئ حساب GitHub** (مجانًا): اذهب إلى
   [github.com/signup](https://github.com/signup)، وأدخل بريدًا إلكترونيًا وكلمة مرور
   واسم مستخدم، وتحقّق من بريدك الإلكتروني. هذا كل شيء.
2. في [Build Studio](/ar/build/)، أكمل الخطوتين 1–2، ثم في **توليد** اختر
   **إنشاء مستودع GitHub**.
3. انقر على **ربط GitHub وإنشاء المستودع**. يطلب منك GitHub التصريح للتطبيق
   بإذن إنشاء مستودع عام — راجِعه ووافق عليه.
4. تعود إلى Studio؛ انقر على **إنشاء**. يُفتح مستودعك الجديد وبه جميع
   ملفات القالب. استنسخه ووجّه وكيلك إليه.

</Steps>

<Aside type="note" title="ما الذي يمكنه الوصول إليه">
لا يطلب التصريح سوى `public_repo` — وهو يكفي لإنشاء مستودع عام واحد
وإضافة الملفات. لا يمكنه المساس بمستودعاتك الخاصة. الرمز قصير الأمد
ويُحتفَظ به فقط للحظة الإنشاء.
</Aside>

## للقائم على الصيانة (إعداد لمرة واحدة)

تحتاج النقرة الواحدة إلى تطبيق OAuth **واحد** من GitHub وسرّين للـ Worker — يُسجَّلان
مرة واحدة، لكامل النشر. هذه هي *هوية* التطبيق (الاسم الذي يراه البانون
على شاشة التصريح في GitHub)، وليست رمز وصول شخصيًّا: فهي ببساطة
تتيح لأي باني التصريح بحسابه الخاص. وإلى أن يُضبط هذان السرّان، يعرض
Studio رسالة "غير مُهيَّأ" ويقدّم ملف `.zip` بدلًا من ذلك.

<Steps>

1. **سجّل تطبيق OAuth:** GitHub → Settings → Developer settings →
   **OAuth Apps → New OAuth App**.
   - Homepage URL: `https://wecanjustbuildthings.dev`
   - Authorization callback URL: `https://wecanjustbuildthings.dev/api/github/callback`
2. انسخ **Client ID** وولّد **Client secret**.
3. أضِفهما بصفتهما سرّين للـ Worker (لوحة Cloudflare ← الـ Worker الخاص بك ← Settings ←
   Variables and Secrets)، أو عبر سطر الأوامر:
   ```sh
   npx wrangler secret put GITHUB_OAUTH_CLIENT_ID
   npx wrangler secret put GITHUB_OAUTH_CLIENT_SECRET
   ```
4. أعِد النشر. سيُبلِّغ `/api/github/status` عن `configured: true` وتُفعَّل النقرة
   الواحدة.

</Steps>

<Aside type="tip" title="لا يوجد إعداد؟ ما زال قابلًا للاستخدام بالكامل">
بدون تطبيق OAuth، تعمل كل طرق التسليم الأخرى: **تنزيل .zip**، و**التشغيل باستخدام
Goose**، و**الانطلاق باستخدام الذكاء الاصطناعي (BYOK)**. النقرة الواحدة وسيلة راحة، لا
شرط.
</Aside>
