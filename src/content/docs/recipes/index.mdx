The exclusion of Meta, OpenAI, and xAI is **absolute**. Recipes are not an
exception to it — they are how the catalog *enforces* it on tools that could
otherwise slip past. Most tools are simply clean or not. A few are *configurable*
to reach an excluded provider at runtime (a bring-your-own-key picker, say); a
recipe **forbids** those providers and machine-verifies they're unreachable.

<Aside type="note" title="A recipe forbids — it never permits">
A recipe must target a catalog entry marked `provider_agnostic: true` (a tool with
a configurable provider picker), name **every** excluded LLM provider in its
`must_not_be_one_of` list, and include verification steps that block the excluded
providers' endpoints. A recipe that fails to forbid OpenAI is **rejected** by CI.
See [how enforcement works](/policies/enforcement/#locking-down-configurable-tools-recipes).
</Aside>

## Proposing a recipe

If you know a catalog tool that can be configured cleanly, propose a recipe via a
[pull request](/contribute/). The recipe validator runs on every PR.
